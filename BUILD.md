## 0. The non-negotiable mindset (from CLAUDE.md; read that file too)

- **Completeness is the goal.** Code + tests + evals + docs, in the same commit. "I'll add tests later"
  is banned.
- **Latent vs deterministic.** Same input, same correct answer by definition = deterministic: write the
  script. Scoring math, quota resets, timezone/con-day windows, synergy bands, DP noise, ledger sums,
  purge sweeps are ALL deterministic space. The only latent surfaces in this product: T9 moderation
  triage, translation, ANIME vector drafts, the AI consultant. Each latent surface gets a prompt + eval;
  everything else gets code + tests.
- **AI/ML access: one router, Claude by default (ruling 15A).** Every model call goes through
  `Svac.AimlRouter` (contract `IAimlRouter`). No provider SDK anywhere else; an architecture test fails
  the build on violation. Provider selection: deterministic routing policy (9A config entry, audited on
  3A) or explicit caller pin. Claude is the default. Dev backend = local Claude Code; prod hot paths call
  provider APIs through the same router, keys in Key Vault.
- **Tie every change to a measurable outcome.** Every slice row in §7 names the metric it moves. The
  Metrics & Ops desk is where the promise is kept.
- **No time estimates, ever.** Deliverables and gates only (founder directive). External calendar facts
  (con dates, store review windows) are permitted as dependencies.
- **Confusion Protocol.** Two plausible architectures, a spec contradiction, a destructive op with
  unclear scope: STOP, name it in one sentence, present 2–3 real options, ask Julien.
